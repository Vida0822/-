1. 인터페이스(interface) 에 대해서 상세히 설명하세요. 
	참조타입의 자료형으로 일종의 추상클래스 역할을 한다
	하지만 추상클래스와 다른점은 필드와 일반 메서드의 선언이 불가능하여
	추상메서드와 상수만을 갖는다
	모든 필드는 public static final이 생략되어있고, 모든 메서드는 public abstract가 생략되어있다
	인터페이스끼리도 extends 키워드를 사용하여 상속이 가능하며 다중상속까지 가능하다
	인터페이스를 구현할 때는 implements 키워드를 사용한다
	인터페이스로 여러 클래스들을 구분지을 수 있다
	인터페이스를 잘 활용하면 개발 시간을 단축시킬 수 있고 표준화가 가능하다
	서로 관련없는 클래스들에게 관계를 맺어줄 수 있다

2. 내부(inner)클래스 중 익명(anonymous)클래스에 대해서 설명하세요. 
	클래스의 선언과 객체의 생성을 동시에 진행하는 클래스이다
	오직 하나의 객체만 생성할 수 있는 일회성 클래스이다
	선언 형식은 두가지가 있다
	new 부모클래스() {
		// 부모클래스 메서드 오버라이딩
	}
	new 인터페이스() {
		// 인터페이스 추상메서드 오버라이딩
	}

3. 예외처리( Exception Handling )에 대해서 설명하세요. 
	프로그램 오류는 컴파일 오류, 런타임 오류, 논리적 오류 이렇게 3가지로 나뉘며
	이 중 런타임 오류는 에러와 예외로 나뉜다
	여기서 예외가 개발자들이 처리할 수 있는 수준의 오류이며 Exception 클래스에 정리되어 있다
	예외처리는 프로그램 실행 시 발생할 수 있는 예외에 대비한 코드를 작성하는 것을 의미하고
	이를 통해 프로그램의 비정상 종료를 막고, 정상적인 실행상태를 유지할 수 있다
	예외처리는 2가지 방법으로 할 수 있다
	try~catch문, throws문
	다중 catch 블럭을 사용할 때 부모 에외 클래스를 가장 밑에 코딩해야한다
